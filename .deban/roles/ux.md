# UX

## Scope
Visual and motion design of the Filter Chamber animation, presentation of the Names section, and tonal coherence between the artifact's subject (institutional moral entropy) and its aesthetic register.

## Decisions

| 2026-05-02 | Animation metaphor: rainbow particles (good will / good intent) and a dark cloud (bad intent) both enter the maelstrom; rainbow particles progressively desaturate and dim as they pass through; output is uniformly dark | Particle desaturation is the load-bearing visual — it makes "good intent passed through wrong system → bad output" legible as motion rather than as static diagram-text. ## Dead Ends
| Date | What was tried | Why it failed / was rejected |
|---|---|---|
| 2026-05-02 | V1 visual: "elegant whirling maelstrom" — a rotating-dashed-circles vortex at center, particles spiraling inward, single dark plume output | Rejected after V1 review against 2016 original. Three issues: (a) the 2016 sketch is a *rectangular chamber* with mechanism names inside, not a vortex — the "maelstrom" framing was Claude's rhetorical overshoot from "filter chamber," and the agent built what was described literally; (b) particles spiraling into a single point loses the "filter" semantics — each mechanism IS a filter, and a particle's loss of color should be cumulative-through-traversal, not collapsed at a vortex center; (c) the rainbow input was too muted in V1 — good/light should be vibrant before it gets diluted. Lesson candidate: when the user gives a primary-source visual reference, the source is canonical; verbal metaphor (mine or theirs) is secondary and may overshoot. |

## Open Questions
- [x] "Animate it elegantly" — is elegance the right register? — resolved 2026-05-02: yes, deliberately. Contrast between elegance and outcome IS the metaphor (see Decisions).
- [x] Aestheticization risk — does it undermine the diagnostic claim? — resolved 2026-05-02: no. Aestheticizing the chamber locates where evil hides; institutions producing the worst outcomes look polished, not crude. The aesthetic is the diagnosis.
- [x] Names section layout — list-of-six vs stack vs progressive disclosure? — resolved 2026-05-02: separate tab, with explicit map-is-not-territory / all-models-wrong meta-framing. See [[pm]] decision on meta-model framing.
- [x] Output rendering — desaturated vs red/violent? — resolved 2026-05-02: desaturated/black. Entropic loss, not added malevolence.
- [x] Particle decay curve — resolved 2026-05-02 (V1.1): linear *per mechanism visit*, but non-deterministic across particles because path length and chosen filters vary. Cumulative effect is roughly exponential because saturation *= 0.85 each visit and visit count varies — encodes "more contact with mechanisms = more drained" without imposing a global decay curve.
- [ ] Tab UI: does the existing kakistogenesis.jsx have a tab structure already, or is "separate tab" a new UI architecture requirement? — owner: Gerald — since: 2026-05-02

2026-05-02 — V1.4.1 chamber typography pass. User reported the 22 mechanism labels and 10 cost labels were "barely legible" on desktop. Audit found the in-SVG `<text>` elements rendering at 11.5 viewBox units each — which scales to ~10–11.5 CSS pixels at typical desktop column widths — and colored `#d8cfb8` (paper) with opacity .88–.9; italic EB Garamond at that size against `#1c1916` background dropped below the WCAG 4.5:1 floor in practice. Hierarchy was also flat: mechanism/cost labels (the primary read) sat at 11.5 while category mono headers were 9.5 — only a 1.2x ratio where mise-en-page principles call for ≥1.5x at adjacent hierarchy levels. Applied a proportional bump across the chamber's typographic register, restraint preserved (no family changes, no sans-serif, no boldface). Mechanism node labels: 11.5 → 14, fill `#ece4d0` (ivory) opacity .95, y-offset 18 → 20 (label sits cleanly below the dot's 7-unit halo). Cost node labels: 11.5 → 14, fill `#ece4d0` opacity .95, y-offset 22 → 24 (matches mechanism labels exactly — peer parity, dice field reads as equal weight to mechanism field). Italic preserved at both: at 14px the contrast loss of italic against dark background is recovered by the brighter ivory and the size bump itself; the museum/book register holds. Left-margin annotation ("Mechanisms / That render / governments / inefficient"): 12.5 → 14.5, color lifted from `--mute` (`#a89e8a`) to `--paper` (`#d8cfb8`), line-spacing widened (16 → 19 units between rows) so the four-line stack reads as a deliberate marginalia voice rather than as faint metadata. Entry-stream captions ("Good intentions" / "Bad intentions"): 14 → 15.5, fill ivory — they now anchor the input columns as the most prominent caption tier in the upper register, which is correct (they label what enters; everything else labels what's processed). Output captions ("Diluted positive outcomes" / "to downright hurtful outcomes"): 13 → 15, fill `--paper`. Footer ("2% of psychopaths ruining it"): 13 → 14 with explicit `fontSize` override on the inner element, color lifted from `--dim` to `--mute` — kept proportionally quieter than the gradient label per the brief's instruction to keep that note understated. Category panel mono headers (SELF-PRESERVATION etc.): 9.5 → 11, opacity .88 → .92, y-offset 22 → 24 — restoring the 1.27x ratio gap between section-anchor type and label type. Die headers (`§ — MATERIAL & HUMAN` / `§ — SOCIAL & EPISTEMIC`): 9.5 → 11, fill `--mute` → `--paper`, opacity .8 → .85, y-offset adjusted to keep clearance of the dashed die frame. Flank labels (DILUTED / HURTFUL): 12 → 13, fill `--paper` → `--ivory` — they were already well-sized per the brief but needed a tiny bump to stay above the new category-header tier. Hierarchy now reads top-down: 15.5 italic captions (input, anchor) → 14.5 italic margin annotation → 14 italic mechanism + cost labels (the load-bearing read) → 13 mono caps flank labels → 11 mono caps section/die headers — clean 1.07–1.27x ratios at adjacent tiers, with italic-vs-mono encoding the semantic axis (prose/voice vs structural/anchor). Collision check: largest mechanism panel has 4 nodes in 2x2 sub-grid with 71-unit slot height; +2.5 viewBox-unit label bump (and label sits 20 below dot center) leaves ~51 units between bottom of top-row label and top of bottom-row dot — generous. Cost dice in wide mode: 64-unit DIE_INSET separates center pip from corner pips; center label at +24 leaves 40 units to the bottom corners' centers, comfortable. Narrow mode (≤768px): same per-element font-sizes apply, but the dice collapse to 2x2 + tucked-center anyway, so vertical space per label is governed by the narrow-mode `inset = 50` — bumped labels still fit. What I did NOT do: (a) did not abbreviate any mechanism or cost names — the hierarchy fix made it unnecessary, and abbreviation would have eroded the "academic register" of the catalogue; (b) did not switch from italic to roman EB Garamond — at 14px the italic reads cleanly, and italic is the load-bearing voice signal that distinguishes the prose labels from the mono structural headers; (c) did not introduce a `--fs-scale` user-facing font-size control because the chamber's labels are SVG-rendered and don't inherit CSS variables cleanly through SVG `<text>` — that would require a state-driven prop pass, out of scope for a typography-only pass; (d) did not touch tab-bar, hero, catalogue/lineage/where-it-works section typography — user surfaced the chamber labels specifically and those other surfaces have their own established hierarchy via `.d-hero`/`.d-xl`/`.d-lg`/`.b-md` classes which are already at appropriate sizes via clamp; (e) did not change geometry — chamber rectangle, panel boundaries, mechanism/cost dot coordinates, particle physics, dice composition, decision-vault entries all preserved. Type-check clean, build clean (vite ✓ 1944 modules, 451 KB), dev server still serves 200. Trade-off acknowledged: the chamber now leans more "readable book" and slightly less "blueprint marginalia" — the museum-shadow restraint holds but the chamber is now an unmistakably-readable diagram first, with the marginalia voice in the supporting tier. That's the right trade for the user's primary complaint (legibility) without sacrificing the aesthetic.
